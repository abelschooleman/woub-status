# Bumblebee-status Module: Unread Messages Count

## Goal
Create a bumblebee-status module that displays the total unread chat message count by calling the existing `GET /api/v1/userschatmob` endpoint with API key authentication.

## API Details (no changes needed)

- **Endpoint:** `GET {base_url}/api/v1/userschatmob`
- **Auth:** `Authorization: ApiKey <key>` header
- **Response:** Array of chat objects, each with a `count_new_messages` integer field
- **Why this endpoint:** Lighter than `/userschat`, excludes archived projects, designed for mobile/lightweight clients

## Module File

**Location:** `~/.config/bumblebee-status/modules/woub.py`

### Implementation

```python
"""Unread messages count from woub API

   Displays the total number of unread chat messages.
   Requires configuration: woub.url and woub.api_key
"""

import core.module
import core.widget
import core.input
import requests


class Module(core.module.Module):
    def __init__(self, config, theme):
        super().__init__(config, theme, core.widget.Widget(self.full_text))

        self.__url = self.parameter("url", "")
        self.__api_key = self.parameter("api_key", "")
        self.__count = 0

        core.input.register(
            core.widget.Widget(self.full_text),
            button=core.input.LEFT_MOUSE,
            cmd="xdg-open {url}".format(url=self.__url.rstrip("/")),
        )

    def full_text(self, widgets):
        if not self.__url or not self.__api_key:
            return "woub: config missing"

        return " {}".format(self.__count) if self.__count else ""

    def state(self, widget):
        return self.threshold_state(self.__count, 5, 15)

    def update(self, widgets):
        try:
            resp = requests.get(
                "{}/api/v1/userschatmob".format(self.__url.rstrip("/")),
                headers={"Authorization": "ApiKey {}".format(self.__api_key)},
                timeout=10,
            )
            resp.raise_for_status()
            chats = resp.json()
            self.__count = sum(
                chat.get("count_new_messages", 0) for chat in chats
            )
        except Exception:
            self.__count = 0
```

### User Configuration

Users pass parameters via CLI or config:

```
bumblebee-status -m woub -p woub.url=https://api.example.com -p woub.api_key=your_api_key
```

Or in `~/.config/bumblebee-status/modules/woub.json`:
```json
{
    "url": "https://api.example.com",
    "api_key": "your_api_key"
}
```

## Key Design Decisions

1. **Endpoint:** `GET /userschatmob` — excludes archived projects, lighter response than `/userschat`
2. **No notification counts** — scope is chat messages only (per user preference)
3. **Widget states:** `warning` at 5+, `critical` at 15+ unread (configurable via theme)
4. **Click behavior:** Opens the woub web URL in browser
5. **Error handling:** Silently shows empty on failure (no crash, no stale data)
6. **Update interval:** Uses bumblebee-status default (user-configurable via `-i`)
